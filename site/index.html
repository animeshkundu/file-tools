Privacy and capability contract

The following wording is the exact capability contract for this extension:

Local processing, no upload; zero install-time permissions for the core; optional downloads permission requested only for tree-preserving extract-all.

This contract is backed by the extension's architecture — all heavy work runs in a Web Worker inside the extension page, nothing is fetched from or sent to any remote host, and the page CSP denies all external connections at the browser level. However, zero manifest permissions alone do not prove privacy: the permission list tells you what the browser would allow, not what the code does. Review the source and the built manifests to verify the contract yourself.

The extension processes files locally and offline. The public Pages site you are reading now is a regular web page served over HTTPS — visiting it uses your network normally.

Permissions breakdown

The extension requests no install-time permissions. When you choose "Download all" to save every extracted entry while preserving the original folder structure, the browser prompts once for the optional downloads permission. You can decline and still download individual files one at a time without any permission.

Safety boundaries

Every ZIP entry passes a set of safety checks before extraction begins:

  • Aggregate emitted bytes, entry count, path depth, and wall time are capped.
  • Path traversal is rejected: entries with .., absolute paths, drive letters, backslashes, NUL bytes, or Windows reserved names are refused.
  • Declared sizes in the ZIP header are treated as untrusted hints.
  • Nested archives are not recursively extracted by default.

If a file exceeds the input size limit or fails any check, extraction stops cleanly and reports the reason. No partial state is left behind.
