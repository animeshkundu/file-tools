# History

Dated records of solved problems, incidents, and non-obvious debugging notes.

Add one file per record, named `YYYY-MM-DD-slug.md`, in the Nygard decision-record
style (Context, Decision/Resolution, Consequences). These complement the
architecture decision records under [`../adr/`](../adr/): ADRs capture forward-looking
decisions; history entries capture what was learned after the fact.
